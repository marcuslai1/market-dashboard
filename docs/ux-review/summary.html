BR-2 / WL-1 / TM-1P1 ✓ Fixed — all 3 surfacesux-fix/br2-* ×2

R:R was shown as the value the data itself flags as distorted

NVDA's headline 46.5:1 comes from a 0.2% stop — the report marks it rr_distorted and even supplies the corrected 4.4:1 under sizing_rr. But the drilldown's corrective “Wide-stop R:R” row read only wide_stop_rr, so it showed “—” and the distorted headline stood alone — even though Terminology promises “both R:R numbers.”
def rr_display(rr): # new shared helper if rr.get("rr_distorted"): # the report's tight-stop flag return rr["sizing_rr"]["ratio_label"], ..., True return rr.get("ratio_label"), ..., False
Now all three surfaces show 4.4:1 — the action card with a “tight-stop adj.” marker, the watchlist column with the raw on hover, the drilldown with both figures — and the “one thing to do” ranking uses the corrected ratio.
BR-1P1 ✓ Fixedux-fix/br1-risk-tag

A plain-sentence risk rendered a truncated fragment instead of a badge

The Active-risks tag did text.split(":")[0][:24], so a normal sentence that happens to contain a colon was sliced mid-word. One of four rows read US‑China tech tensions p where the others showed a clean severity badge. Now the tag is only used when it's genuinely short and tag-like; otherwise it falls back to the severity — all four rows show LOW. (41 real “Category:” prefixes in the corpus still qualify.) +2 regression tests.
RC-1P2 ✓ Fixedux-fix/rc1-display-ticker

Foreign tickers showed raw keys — AIXA_DE instead of AIXA.DE

Report Comparison was the only page that never applied display_ticker(), so it printed the sanitized watchlist keys. The five .DE/.SI/.KS/.PA listings showed with underscores — symbols that don't exist. One-line-per-site wrap; cosmetic, zero risk.
ST-1P2 ✓ Fixedux-fix/st1-tracker-coloring

Correct CAUTION calls were painted red on the honest-calibration page

The by-name ledger coloured returns by raw sign — but a CAUTION call is “right” when price falls. So NVTS’s -53.1% (a great avoid) rendered red, while INTC’s +6.8% (a wrong-so-far avoid) rendered green — inverted, and today’s mostly-CAUTION list read as a “sea of red” that was mostly correct. You’d already solved this on the calibration cards; the ledger just wasn’t brought in line.
- style="color:{_ret_color(v)}" # sign-based red / green + (neutral) # +/- sign still shows direction
Returns are now neutral; the direction-aware “Trades won” winbar carries performance.
