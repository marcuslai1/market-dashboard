BR-2 / WL-1 / TM-1P1 ✓ Fixedux-fix/br2-drilldown-sizing-rr

R:R was shown as the value the data itself flags as distorted

NVDA's headline 46.5:1 comes from a 0.2% stop — the report marks it rr_distorted and even supplies the corrected 4.4:1 under sizing_rr. But the drilldown's corrective “Wide-stop R:R” row read only wide_stop_rr, so it showed “—” and the distorted headline stood alone — even though Terminology promises “both R:R numbers.”
wide_stop = rr_obj.get("wide_stop_rr") # was: "—" for NVDA + if wide_stop is None: + wide_stop = (rr_obj.get("sizing_rr") or {}).get("ratio") # → 4.40:1
Now the corrected ratio sits right beside the headline, as intended.
BR-1P1 ✓ Fixedux-fix/br1-risk-tag

A plain-sentence risk rendered a truncated fragment instead of a badge

The Active-risks tag did text.split(":")[0][:24], so a normal sentence that happens to contain a colon was sliced mid-word. One of four rows read US‑China tech tensions p where the others showed a clean severity badge. Now the tag is only used when it's genuinely short and tag-like; otherwise it falls back to the severity — all four rows show LOW. (41 real “Category:” prefixes in the corpus still qualify.) +2 regression tests.
RC-1P2 ✓ Fixedux-fix/rc1-display-ticker

Foreign tickers showed raw keys — AIXA_DE instead of AIXA.DE

Report Comparison was the only page that never applied display_ticker(), so it printed the sanitized watchlist keys. The five .DE/.SI/.KS/.PA listings showed with underscores — symbols that don't exist. One-line-per-site wrap; cosmetic, zero risk.

Ranked proposals

Design judgment, not defects. Your UI choices are deliberate, so these stay proposals — ranked by impact.

  1. BR-2, the headline half P2
    The action card & watchlist column still headline the distorted 46.5:1. Whether to show 4.4:1 there, show both, or add a “tight-stop” marker is an editorial call. Suggested: prefer sizing_rr when rr_distorted, with a small marker.
  2. ST-1 — Signal Tracker returns are coloured by raw sign P2
    A correct CAUTION call (price fell) renders red; a wrong one (rose) renders green — inverted vs. the call being right. You already neutralised exactly this on the calibration cards; the by-name ledger wasn't brought in line.
  3. SC-1 — “Wildcard” is a different colour across pages P3
    Amber on the Briefing odds bar, purple on the Scenario Log. May be a deliberate colour-blind choice — align if not.
  4. BR-4 — the posture verdict is one ~60-word <h2> P3
    Leading with the verdict is great; as a heading it bloats the screen-reader outline. A styled <p> + a short real heading keeps the look, fixes the semantics.
